# Codify

> 개인 보유 의류 데이터화 및 맥락 인지형(TPO/날씨) 데일리 코디 추천 서비스

---

## 📋 프로젝트 개요

### 배경 (Problem)

"옷장은 꽉 찼는데 오늘 입을 옷은 없다"는 역설은 현대인의 대표적인 '결정 피로(Decision Fatigue)' 사례입니다. 많은 이들이 자신이 가진 옷의 종류를 정확히 파악하지 못해 중복 구매를 하거나, 매일 아침 기온과 상황(TPO)에 맞는 조합을 찾는 데 불필요한 에너지를 소모하고 있습니다.

### 목적 (Solution)

본 프로젝트는 사용자의 옷장을 디지털 데이터베이스로 전환하여 자산 관리의 편의성을 높이고, LLM을 활용해 "격식 있지만 너무 딱딱하지 않은"과 같은 복합적인 요구사항을 실제 날씨 데이터와 매칭하여 해결하는 것을 목표로 합니다. 단순히 옷을 골라주는 것을 넘어, 사용자의 '결정 비용'을 제로로 만드는 데 집중합니다.

### 앱 컨셉 및 디자인 철학

**테마**: "프리미엄 퍼스널 스타일리스트"

- **비주얼**: 다크 모드 기반, 글래스모피즘 효과(반투명 카드), 부드러운 진입 애니메이션
- **톤앤매너**: 전문적이며 격려하는 어조, 스마트한 사용자 경험
- **목표**: 상황(날씨/일정)과 옷장 데이터를 결합하여 옷 입기 결정 피로를 해결

## 🛠 기술 스택

### 엔진 (LLM & Search)
- **GPT-4o**로 자연어 요청을 TPO·제약으로 구조화
- 사용자 데이터·임베딩 기반 **Vector DB** 검색
    - 대화 기록 기반 사용자의 취향 분석
- 규칙·가중치로 코디 랭킹
    - ML 고도화하면 좋음

### 프론트엔드
- **Flutter**로 디지털 옷장 UI 구현
- 다크 모드 테마, 글래스모피즘 효과, 부드러운 애니메이션
- 상품 스캔·TPO 입력·추천 결과 노출
- 하단 탭 네비게이션 및 스택 네비게이션

### 백엔드
- **FastAPI**로 REST API 구축
- **ChromaDB** 비동기 처리
- **LangGraph** 활용 LLM 처리
- 이미지 업로드 처리

### 클라우드 아키텍처
- **Azure Load Balancer**·**VNet**·**Private Endpoint**로 보안 강화
- **Postgres VectorDB**로 비용 1/3 절감
- **SignalR**로 AI 분석 UX 개선

## 🎯 핵심 기능 (MVP)

### A. 지능형 홈 대시보드
- **오늘의 픽**: AI가 선택한 최적의 코디를 큰 카드로 표시
- **인터랙티브**: 마음에 들지 않으면 새로고침하여 다른 추천 받기
- **AI 채팅**: 특정 요청(예: "오늘 데이트 있어")을 위한 채팅 인터페이스
- **상황 인식**: 현재 날씨와 일정을 표시하여 추천의 근거를 제시
- **상세 정보**: 카드 탭 시 매칭 점수와 추천 사유 확인

### B. 코디 상세 보기
- **탐색**: 추천받은 코디를 개별 아이템(상의, 하의)으로 분해하여 확인
- **투명성**: 매칭 점수와 텍스트 설명을 통해 신뢰 구축
- **액션**: 캘린더에 저장 기능

### C. 디지털 옷장
- **아이템 추가**: 사진 선택 → AI 분석 시뮬레이션 → 속성 자동 태깅(카테고리, 색상)
- **탐색**: "상의", "하의" 등으로 필터링 가능한 옷장 그리드 뷰
- **통계**: 신체 프로필 데이터 및 착용 통계 표시

### D. 아이템 상세 보기
- **메타데이터**: 사용 통계(착용 횟수, 마지막 착용일) 표시
- **속성**: 카테고리, 색상, 브랜드 정보 관리

## 🔄 사용자 워크플로우

### 워크플로우 1: "외출 준비" (아침 루틴)
1. 앱 실행: 사용자 맞춤 인사말과 상황 정보(날씨/이벤트) 확인
2. 추천 확인: 시스템이 "오늘의 픽"을 제시
3. 결정:
   - 마음에 들면: 카드 탭 → 개별 아이템 확인 → 착용
   - 마음에 안 들면: 새로고침 아이콘 탭 → 새로운 옵션 제시

### 워크플로우 2: "옷장 채우기" (온보딩/관리)
1. 시작: 홈 화면이나 옷장 탭의 (+) 버튼 탭
2. 촬영/선택: 갤러리에서 사진 선택 또는 카메라 촬영
3. 처리: AI 분석 진행바 표시 → 자동 태깅 완료
4. 확인: 자동 감지된 태그 확인 (예: "검정 티셔츠")
5. 저장: 완료 탭 → 옷장 그리드에 아이템 추가

### 워크플로우 3: "특별한 이벤트" (채팅 or 음성)
1. 접근: 홈 화면 채팅 섹션으로 이동
2. 요청: 요청 사항 입력 (예: "격식 있는 저녁 식사 코디해줘")
3. 수신: AI가 해당 요청에 맞춰 새로운 코디 큐레이팅

## ✨ 기대효과

### 일상의 효율화
아침 시간 코디 결정 과정을 자동화하여 일평균 10~15분의 준비 시간을 단축하여 개인의 피로도를 줄일 수 있습니다.

### 합리적 소비 유도
보유 의류의 디지털 가시성을 확보함으로써 유사 아이템의 중복 구매를 방지하고, 잠자고 있는 옷의 활용도를 극대화합니다.

### 패션 데이터 자산화
개인의 스타일 선호도와 착용 빈도를 데이터로 축적하여, 향후 퍼스널 쇼퍼나 의류 정리 서비스로 확장할 수 있는 기반을 마련합니다.

## 🔒 책임있는 인공지능 (윤리 및 안전성)

### 데이터 프라이버시
개인의 의류 및 체형 정보는 특징점(Vector) 형태로만 관리하며, 민감한 전신 사진 데이터는 서버 저장 최소화 및 비식별화 처리를 원칙으로 합니다.
- VectorDB에 적재된 정보를 훔쳐가도 디코딩 할 수 없으면 크게 리스크가 없음

### 추천의 투명성 (XAI)
"오늘 오후 비 소식이 있어 기능성 외투를 포함했습니다"와 같이 AI가 해당 코디를 제안한 근거를 사용자에게 함께 제시하여 신뢰성을 높입니다.

### 기술적 안전성
날씨 데이터 오류 등으로 인한 부적절한 추천을 방지하기 위해 기온별 최소/최대 방한 지수를 가이드라인으로 설정하여 안전성을 검증합니다.
