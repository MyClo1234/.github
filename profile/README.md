# Codify

> 개인 보유 의류 데이터화 및 맥락 인지형(TPO/날씨) 데일리 코디 추천 서비스

---

## 📋 프로젝트 개요

### 배경 (Problem)

"옷장은 꽉 찼는데 오늘 입을 옷은 없다"는 역설은 현대인의 대표적인 '결정 피로(Decision Fatigue)' 사례입니다. 많은 이들이 자신이 가진 옷의 종류를 정확히 파악하지 못해 중복 구매를 하거나, 매일 아침 기온과 상황(TPO)에 맞는 조합을 찾는 데 불필요한 에너지를 소모하고 있습니다.

### 목적 (Solution)

본 프로젝트는 사용자의 옷장을 디지털 데이터베이스로 전환하여 자산 관리의 편의성을 높이고, LLM을 활용해 "격식 있지만 너무 딱딱하지 않은"과 같은 복합적인 요구사항을 실제 날씨 데이터와 매칭하여 해결하는 것을 목표로 합니다. 단순히 옷을 골라주는 것을 넘어, 사용자의 '결정 비용'을 제로로 만드는 데 집중합니다.

## 🛠 기술 스택

### 엔진 (LLM & Search)
- **GPT-4o**로 자연어 요청을 TPO·제약으로 구조화
- 사용자 데이터·임베딩 기반 **Vector DB** 검색
    - 대화 기록 기반 사용자의 취향 분석
- 규칙·가중치로 코디 랭킹
    - ML 고도화하면 좋음

### 프론트엔드
- **Flutter**로 디지털 옷장 UI 구현
- 상품 스캔·TPO 입력·추천 결과 노출

### 백엔드
- **FastAPI**로 REST API 구축
- **ChromaDB** 비동기 처리
- **LangGraph** 활용 LLM 처리
- 이미지 업로드 처리

### 클라우드 아키텍처
- **Azure Load Balancer**·**VNet**·**Private Endpoint**로 보안 강화
- **Postgres VectorDB**로 비용 1/3 절감
- **SignalR**로 AI 분석 UX 개선

## ✨ 기대효과

### 일상의 효율화
아침 시간 코디 결정 과정을 자동화하여 일평균 10~15분의 준비 시간을 단축하여 개인의 피로도를 줄일 수 있습니다.

### 합리적 소비 유도
보유 의류의 디지털 가시성을 확보함으로써 유사 아이템의 중복 구매를 방지하고, 잠자고 있는 옷의 활용도를 극대화합니다.

### 패션 데이터 자산화
개인의 스타일 선호도와 착용 빈도를 데이터로 축적하여, 향후 퍼스널 쇼퍼나 의류 정리 서비스로 확장할 수 있는 기반을 마련합니다.

## 🔒 책임있는 인공지능 (윤리 및 안전성)

### 데이터 프라이버시
개인의 의류 및 체형 정보는 특징점(Vector) 형태로만 관리하며, 민감한 전신 사진 데이터는 서버 저장 최소화 및 비식별화 처리를 원칙으로 합니다.
- VectorDB에 적재된 정보를 훔쳐가도 디코딩 할 수 없으면 크게 리스크가 없음

### 추천의 투명성 (XAI)
"오늘 오후 비 소식이 있어 기능성 외투를 포함했습니다"와 같이 AI가 해당 코디를 제안한 근거를 사용자에게 함께 제시하여 신뢰성을 높입니다.

### 기술적 안전성
날씨 데이터 오류 등으로 인한 부적절한 추천을 방지하기 위해 기온별 최소/최대 방한 지수를 가이드라인으로 설정하여 안전성을 검증합니다.
